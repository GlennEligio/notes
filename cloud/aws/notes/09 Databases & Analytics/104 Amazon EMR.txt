104 Amazon EMR

Amazon EMR
 > EMR stands for "Elastic MapReduce"
 > EMR helps creating Hadoop clusters (Big Data) to analyze and process vast amount of data
 > The Hadoop clusters can be made of hunders of EC2 instances
 > Also supports Apache Spark, HBase, Presto, Flink...
 > EMR takes care of all the provisioning and configuration of these Hadoop cluster so that they can work and analyze together
 > Auto-scaling and integrated with Spot instances
 > Use cases include:
	- data processing
	- machine learning
	- web indexing
	- big data


NOTES:
1. EMR is not a database
