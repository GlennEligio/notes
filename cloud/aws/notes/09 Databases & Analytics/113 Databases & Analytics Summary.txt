Relation Databases - OTLP: RDS & Aurora (SQL)
 > OTLP - Online

Differences between Multi-AZ, Read Replicas, Multi-Region
 > Multi-AZ, database situated in mutli-AZ, composed of Main DB and Failover DB to be used as backup
 > Read Replicas, database replicas for read operations only, for spreading read workload
 > Multi-Region, mutiple read-only database replicas all over different region, write operation is still on main database

In-memory Database: ElastiCache
