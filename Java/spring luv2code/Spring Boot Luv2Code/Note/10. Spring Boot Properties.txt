10. Spring Boot Properties

Spring Boot Properties
	> we can configure Spring Boot in application.properties file
	> these properties can be roughly grouped up into the following categories
		1. Core
		2. Web
		3. Security
		4. Data
		5. Actuator
		6. Integration
		7. DevTools
		8. Testing


Core
	> in Core, we can include the following
		- Logging
		- MessagingQueue
	> some of the properties include
		- Setting up log levels of specific package names
			= logging.level.package.name
		- Setting up location of logging data
			= logging.file=my-crazy-stuff.log


EX:
# Log levels severity mapping for each package names
logging.level.org.springframework=DEBUG
logging.level.org.hibernate=TRACE
logging.level.com.luv2code=INFO


Logging levels
	- from the less verbose to more verbose (except OFF), here are the list of logging levels available
		1. TRACE
		2. DEBUG
		3. INFO
		4. WARN
		5. ERROR
		6. FATAL
		7. OFF



Web
	> related to Spring MVC
	> these includes
		a. Setting up Http server port to listen
		b. Setting up context path of application
		c. Defining session time out duration of application

EX:
# HTTP server port
server.port=7070

# Context path of the application
server.servlet.context-path=/my-silly-app

# Default HTTP session time out
server.servlet.session.timeout=15m




Actuator
	> related to Spring Boot Actuator
	> may include
		a. Defining endpoints to expose
		b. Defining endpoints to exclude the exposure
		c. Setting up base path for actuator endpoints

EX:
# Endpoints to include by name or wildcard
management.endpoints.web.exposure.include=*

# Endpoints to exlude by name or wildcard
management.endpoints.web.exposure.exclude=beans,mapping

# Base path for actuator endpoints
management.endpoints.web.base-path=/actuator



Security
	> related to Spring Security
	> may include
		a. Defining username and password of default login page

EX:
# Default user name
spring.security.user.name=admin

# Password for default user
spring.security.user.password=topsecret



Data Properties
	> related to Spring Boot Data JPA
	> may include
		a. Setting up properties of DataSource bean like
			- DB_URL
			- username
			- password

EX:
# JDBC URL of the database
spring.datasource.url=jdbc:mysql://localhost:3306/ecommerce

# Login username of the database
spring.datasource.username=scott

# Login password of the database
spring.datasource.password=tiger
